Comentários em Python

- Comentários são importantes para tornar seu código mais legível e compreensível.

- Em Python, comentários são criados usando o símbolo #.

- Tudo o que estiver após o # será ignorado pelo interpretador Python.

# Este é um comentário em Python
print("Olá, Mundo!") # Esta linha imprime uma mensagem

- Comentários podem ser usados para explicar variáveis e operações.

x = 10 # Define a variável x com valor 10
y = 20 # Define a variável y com valor 20
z = x + y # Soma x e y e armazena o resultado em z

Comentários de Múltiplas Linhas

- Para comentários mais longos, você pode usar comentários de múltiplas linhas.

- Embora Python não tenha um símbolo específico para isso, você pode usar múltiplos #.

# Este é um comentário
# de múltiplas linhas
# explicando o próximo bloco de código

- Alternativamente, você pode usar strings de múltiplas linhas (docstrings) como comentários.

"""
Este é um comentário de
múltiplas linhas usando
uma string de múltiplas linhas.
"""

Boas Práticas de Comentários

- Mantenha seus comentários claros e concisos.

- Comente o "porquê" e não o "como".

- Use comentários para explicar partes complicadas do código.

- Evite comentários óbvios que não agregam valor.

# Calcula a área do círculo
def calcula_area(raio):
    return 3.14 * raio ** 2
Comentários ajudam a documentar seu código, mas devem ser usados com moderação e sabedoria.

Comentários e Docstrings

- Docstrings são strings de documentação usadas para explicar o propósito de uma função, classe ou módulo.

- Elas são colocadas logo abaixo da definição da função, classe ou módulo.

def minha_funcao():
    """Esta função faz algo muito importante."""
    print("Executando a função")
Docstrings são acessíveis via a função help() e são uma excelente prática para documentar seu código.
